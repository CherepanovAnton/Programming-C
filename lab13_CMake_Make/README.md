# Make
**Make** - утилита для автоматизации процесса сборки программного обеспечения с помощью специального файла `Makefile`. Часто используется для проектов на C/C++, также можно встретить Makefile для выполнения команд в терминале (например при работе с docker). Понимает зависимости между файлами и собирает по уму, выбирая только измененные файлы                             

make - утилита и ее необходимо установить:
```bash
sudo apt update
sudo apt install build-essential -y
```

**Makefile** — это файл, который хранится обычно в корне проекта. 
В общем виде синтаксис makefile можно представить так:  
```
цель1: реквизит1 реквизит2  
	команда1 # Важно: для отступа используется табуляция
	команда2 # команды будут выполняться последовательно и только в случае успеха предыдущей  
```
Важно: По умолчанию make ищет `Makefile`, если он у вас называется как-то иначе, то через `-f` ключ необходимо указать путь:
```bash
make -f MyMakefile
```

# CMake
Установить CMake:               
```bash
sudo apt update
sudo apt install build-essential cmake -y 
```
**CMake** - кроcсплатформенная утилита для автоматической сборки программы из исходного кода. Если Make - это список команд для компилятора, то CMake - это инструмент более высокого уровня. Он не заменяет Make, а стоит над ним. Он работает с файлами по фамилии `CMakeLists.txt`, в которых на языке CMake описывается конфигурация и сборка проекта. Далее CMake генерит готовый к использованию Makefile

**Важно:** CMake не собирает проект, а лишь конфигурит и генерит Makefile, который уже собирает проект.


Все временные файлы живут в папке `build`, которую можно удалить одной командой, не боясь зацепить важный код. Классический способ собрать типовой проект, использующий CMake (внутри склонированного проекта):
```bash
mkdir build && cd build
cmake ..
make
```
- Создаем каталог build и заходим в него
- Вызываем утилиту cmake для генерации Makefile и прописываем через пробел путь до CMakeLists.txt (.. - две точки означает на каталог выше, т.е. в предыдущей папке)
- Собираем проект с помощью make         

Эта инструкция из нескольких шагов встречается повсеместно и ее необходимо запомнить и понять            

Либо тоже самое двумя командами, используя лишь cmake (так нынче модно и современно):   
```bash
cmake -B build
cmake --build build
```

**Важно:** **CMake** вызывается единожды для конфигурации Makefile, далее при изменении исходник мы вызываем только **make**! **CMake** повторно имеет смысл вызывать если что-то меняется в самом CMakeLists.txt
